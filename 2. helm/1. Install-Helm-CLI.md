## üìã Installation Methods

### Windows (using Chocolatey)
```bash
choco install kubernetes-helm
```

### Windows (using Scoop)
```bash
scoop install helm
```

### Windows (Manual Download)
1. Go to [Helm Releases](https://github.com/helm/helm/releases)
2. Download the Windows binary
3. Extract and add to PATH

### macOS (using Homebrew)
```bash
brew install helm
```

### Linux (using Snap)
```bash
sudo snap install helm --classic
```

### Linux (using Package Manager)
```bash
# Ubuntu/Debian
curl https://baltocdn.com/helm/signing.asc | gpg --dearmor | sudo tee /usr/share/keyrings/helm.gpg > /dev/null
sudo apt-get install apt-transport-https --yes
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/helm.gpg] https://baltocdn.com/helm/stable/debian/ all main" | sudo tee /etc/apt/sources.list.d/helm-stable-debian.list
sudo apt-get update
sudo apt-get install helm
```

## ‚úÖ Verification Commands

### Check Helm Version
```bash

```

Expected output:
```

```

### Check Helm Help
```bash

```

### Verify Kubernetes Connection
```bash
helm list
```

## üîß Initial Setup

### Add Official Helm Chart Repository
```bash
helm repo add stable https://charts.helm.sh/stable
helm repo add bitnami https://charts.bitnami.com/bitnami
```

### Update Repository Information
```bash
helm repo update
```

### List Available Repositories
```bash
helm repo list
```

## üö® Common Installation Issues

### Issue: "helm: command not found"
**Solution**: Add Helm binary to your system PATH

### Issue: "connection refused" when running helm commands
**Solution**: Ensure kubectl is configured and can connect to your cluster
```bash
kubectl cluster-info
```

### Issue: Permission denied on Linux/macOS
**Solution**: Make the helm binary executable
```bash
chmod +x /usr/local/bin/helm
```

## üéì Key Takeaways

- Helm CLI is required for managing Kubernetes applications
- Multiple installation methods available for different operating systems
- Verify both Helm installation and Kubernetes connectivity
- Repository management is essential for accessing charts
- Helm v3 doesn't require Tiller (server-side component)
